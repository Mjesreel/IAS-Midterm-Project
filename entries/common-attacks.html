Common Attacks

Understanding how attacks occur to build better defenses.

Attacks

What I Learned

Learning about common attacks was eye-opening. I realized that most successful attacks aren't sophisticatedâ€”they exploit simple mistakes and common vulnerabilities. Understanding these attacks helps me think about security more practically.

Phishing

Phishing is when attackers trick people into giving up sensitive information. They might send an email that looks like it's from your bank, asking you to click a link and enter your password. The link takes you to a fake website that steals your credentials.

What I learned: People are often the weakest link in security. No matter how strong your technical defenses are, if someone can be tricked into handing over their password, the attacker gets in.

Brute Force Attacks

A brute force attack is when an attacker tries many different passwords until they find the right one. It's like trying every key on a keyring until one opens the door.

Modern tools can try thousands of passwords per second. That's why using strong, unique passwords is so important. A password like "password123" can be cracked in seconds. A long, random password might take years.

SQL Injection

SQL injection happens when an attacker inserts malicious code into a web form or URL. If the website doesn't properly validate input, the attacker's code gets executed by the database.

For example, instead of entering a username, an attacker might enter: ' OR '1'='1. This can trick the database into returning all user records or even giving the attacker admin access.

Privilege Escalation

Privilege escalation is when an attacker gains higher-level access than they should have. They might start as a regular user but find a way to become an administrator.

This often happens through misconfigured permissions, unpatched software, or exploiting vulnerabilities in how the system handles user privileges.

How to Defend

For each attack, I learned practical defenses:

  • Phishing: User education, email filtering, multi-factor authentication
  • Brute Force: Strong passwords, account lockouts, rate limiting
  • SQL Injection: Input validation, parameterized queries, least privilege
  • Privilege Escalation: Proper permissions, regular patching, monitoring
"Understanding how attacks work is the first step to preventing them. Every attack teaches you something about how to build better defenses."
