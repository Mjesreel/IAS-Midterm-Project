Linux Fundamentals Part 2

Deeper into file systems, processes, and system administration.

Linux

What I Learned

Building on the basics, Part 2 took me deeper into how Linux actually works under the hood. I learned about file systems, how processes run, and the basics of system administration.

Understanding File Systems

Linux treats everything as a file—even hardware devices. I learned about the directory structure and what each folder is for:

  • /etc - Configuration files
  • /var - Variable data like logs
  • /home - User home directories
  • /tmp - Temporary files
  • /bin and /usr/bin - Executable programs

Working with Processes

I learned how to view and manage running processes using commands like ps, top, and kill. Understanding processes is crucial because malware often runs as a process, and knowing how to identify and stop suspicious processes is a key security skill.

System Administration Basics

I got hands-on experience with:

  • Managing users and groups
  • Installing and updating software with package managers
  • Checking system logs for errors or suspicious activity
  • Understanding system services and how they start

Why This Matters

Most servers run Linux. Whether you're attacking or defending, you need to understand how the system works. Knowing where logs are stored, how to check running processes, and how permissions work gives you the foundation to do real security work.
