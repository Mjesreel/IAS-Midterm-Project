CherryBlossom

Weak cryptography leads to full compromise.

Cryptography

What I Learned

CherryBlossom was my introduction to cryptography and password cracking. I learned that encryption is only as strong as its implementation—and weak crypto can be broken surprisingly easily.

Understanding Encryption

Encryption is supposed to protect data by scrambling it so only authorized people can read it. But if the encryption is weak or poorly implemented, attackers can break it.

I learned about different types of encryption:

  • Symmetric encryption: Same key for encryption and decryption (like AES)
  • Asymmetric encryption: Public and private key pairs (like RSA)
  • Hashing: One-way functions used for passwords (like SHA-256, bcrypt)

Password Cracking

In this challenge, I used tools like hashcat and john to crack password hashes. I learned that:

  • Weak passwords can be cracked in seconds
  • Dictionary attacks use common passwords and words
  • Brute force tries every possible combination
  • Rainbow tables are pre-computed hash databases

Key Management

One of the biggest lessons was about key management. Even strong encryption is useless if:

  • Keys are stored insecurely
  • Keys are too short or predictable
  • Keys are reused across multiple systems
  • Keys are never rotated or updated

Why This Matters

Cryptography protects everything from passwords to financial transactions to private communications. Understanding how it works—and how it can fail—is essential for both attackers and defenders. Weak crypto is one of the most common vulnerabilities in real-world systems.

"Encryption is not magic. It's math. And if the math is weak or the implementation is flawed, the protection crumbles."
